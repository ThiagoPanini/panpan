# AGENTS.md — panpan portfolio

Personal portfolio of **Thiago Panini** (Staff Analytics Engineer). Astro static site deployed to Vercel at https://thiagopanini.dev. Zero client-side JS by default; GSAP animations are the only deliberate exception.

## Architecture

```
src/
  components/   # Astro components — PascalCase (Hero.astro, Nav.astro, Projects.astro, AIStack.astro, AIUsage.astro, CoverageCard.astro)
  layouts/      # Page wrappers (BaseLayout.astro)
  pages/        # File-based routes (index.astro)
  styles/       # Vanilla CSS — kebab-case filenames, custom properties as design tokens
  scripts/      # Client-side TypeScript (animations.ts, neuralNetwork.ts)
  data/         # Auto-generated JSON — never edit manually
  assets/       # Optimized assets processed by Astro
public/         # Static assets (favicon, images)
scripts/        # Node.js automation (generate-digest.mjs, generate-coverage.mjs, generate-languages.mjs)
.github/
  workflows/    # deploy.yml (Vercel), daily-update.yml (GitHub data refresh), feature.yml, security.yml
```

Path alias: `@/*` resolves to `src/*`.

## Dependencies

- **astro** — static site generator
- **gsap** — scroll/entrance animations (only client-side JS)
- **@vercel/analytics** — page-view tracking
- **typescript** (dev) — type checking for client scripts

## Commands

```bash
npm run dev      # dev server → http://localhost:4321
npm run build    # production build → dist/
npm run preview  # preview dist/ locally
```

No test or lint scripts are configured. `npm run build` is the only validation gate.

## Conventions

- **Components**: `.astro` files, PascalCase names.
- **CSS**: Vanilla CSS, kebab-case filenames, custom properties for design tokens. No Tailwind.
- **Scripts (client)**: TypeScript under `src/scripts/`. Loaded with `<script>` tags only when explicitly needed.
- **Scripts (automation)**: ES modules (`.mjs`) under `scripts/`. Node.js only, not bundled by Astro.
- **Data files** (`src/data/`): auto-generated by GitHub Actions daily update — do not edit manually.
- **Imports**: use the `@/` alias for `src/`-relative imports inside Astro/TS files.
- **Astro islands**: avoid `client:*` directives unless interactivity is explicitly required.
- **No framework JS** (React, Vue, Svelte) is used; keep it that way unless the user explicitly requests it.

## Design tokens

All design tokens are defined as CSS custom properties in `src/styles/global.css`:

- Colors: `--color-bg`, `--color-bg-subtle`, `--color-text-primary`, `--color-text-secondary`, `--color-accent`, `--color-accent-hover`, `--color-border`
- Typography: `--font-display` (Space Grotesk), `--font-body` (Outfit), `--text-display`, `--text-overline`, `--text-body`, `--text-cta`, `--text-nav`, `--text-small`
- Spacing: `--space-xs` through `--space-3xl`
- Layout: `--max-width`, `--padding-x`
- Transitions: `--ease-out`, `--ease-in-out`

Always use these tokens instead of hard-coded values.

## Agent scope

- Make changes only to `src/`, `scripts/`, `.github/workflows/`, and root config files.
- Do not modify `dist/`, `.astro/`, `node_modules/`, or `.vercel/`.
- Do not edit files under `src/data/` — they are machine-generated.
- Do not add npm dependencies without explicit user approval.
- Do not create new GitHub Actions workflows; amend existing ones only when asked.
